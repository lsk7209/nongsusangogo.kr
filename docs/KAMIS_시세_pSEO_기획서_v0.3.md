## 0. 한 줄 요약
KAMIS 일별 도·소매 시세를 **시계열 누적**해 "언제 싸나·제철 저점·평년 대비·대체재"를 **도구로 즉답**하는 식품·생활물가 사이트. 즉답형("오늘 가격")은 제로클릭으로 죽으므로 시간·계절·대체 의도가 메인. **단, 두 개의 이진 kill 변수(①지역 차원 ②상업적 재배포 라이선스)가 모두 풀려야 빌드 가능.**

## 1. 🚨 두 개의 이진 Kill 게이트 (v0.3 최상위)

이 컨셉은 빌드 전에 **반드시 둘 다 통과**해야 한다. 하나라도 막히면 즉사 또는 중대 축소.

### Kill 게이트 ① — 지역 차원 (페이지 양산 가능성)
무료 Open API 일별 응답에 `mrkt_nm`/`ctnp_nm`/`sggu_nm` 지역 차원이 내려오는가.
→ 3갈래: **A-free**(무료에 있음) / **A-paid**(KADX 유료에만) / **B**(없음, 도구·시계열 축소생존).

### Kill 게이트 ② — 상업적 재배포 라이선스 [신규·치명]
KAMIS 데이터의 **영리 목적 대량 재배포**가 약관·공공누리 유형상 허용되는가.
- 공공누리 **제1유형**(출처표시)이면 상업 이용 OK → 통과.
- **제2~4유형**(상업 금지/변경 금지 등)이면 → **지역 차원과 무관하게 컨셉 전체 kill** 또는 비영리 전환.
- **KADX 유료 데이터는 돈을 내도 재배포권이 더 좁을 수 있음** → A-paid는 ROI 이전에 **라이선스부터** 확인. 재배포 불가면 A-paid 자동 탈락.
- 프로브에 KAMIS 이용약관·KADX 데이터상품 라이선스 조항 확인을 **반드시 포함.**

> v0.1~0.2는 라이선스를 "출처표시"로만 다뤘으나, 실제로는 **지역 차원과 동급의 생사 변수**다.

## 2. v0.2 대비 변경 요약 (5인 리뷰 결론)
1. **라이선스를 2차 kill 게이트로 승격** (§1)

6. **프로그매틱 vs 에디토리얼 URL 분리** (시세 팩트 페이지 ↔ AI작성 시즌 허브) (§5)

## 4. 데이터 모델 & 정규화 (함정 집약)

### 4.1 단위(`unit`) [최우선·minSal 동형]
무게 기반 vs **개수 기반 kg환산 불가**(포기·구·손·마리·장). `unit_type` enum 명시, **전수 분류 확정 전 정규화 코드 금지.**

### 4.2 소매↔도매 조인
`pdlt_code`+`spcs_code` 1:1 + 환산가능 품목 한정 마진. 프로브 매칭률 측정.

### 4.3 평년·전년·전월 정의
5일 이동평균 / 평년=5년 중 최고·최저 제외 3년 평균 → 카피 각주 필수.

### 4.4 할인가 플래그
멤버십·정부지원 할인 혼입 → 평시 시세 왜곡. 플래그 유무 프로브.

### 4.5 빈 셀(`-`) sparse
결측 보간 금지, "데이터 없음" 명시, %변동 시 결측 제외.

### 4.6 등급(rank) 차원 [신규]
- 같은 품목이 **상/중/하 등급별 다른 가격** → `price_daily`가 (품목×품종×**등급**×도소매×지역×날짜)로 폭발.
- **헤드라인 정책 필요**: 기본 표시 등급(예: 중품) 1개를 대표값으로, 나머지는 펼침. "어느 등급 가격인지" 페이지에 명시(오정보 방지).
- Turso 용량 추산을 게이트 0.5에 추가(행수 = 위 차원 곱 × 시계열 일수).

### 4.7 스키마
```
item_meta   — unit, unit_type, weight_g, is_discount_capable, category
price_daily — date, item_code, kind_code, rank, wsrt, region_code(A경로),
              price, price_per_kg(nullable), is_discount, prev_day, m1_ma5, y1_ma5, normal_3yr
price_monthly / price_yearly — stitching
substitutes — item_code, sub_item_code, relation(대체/보완)   # 신규
season_calendar — event, lead_weeks, target_items[], target_hubs[]
pages, fillers, evergreens, hubs, publish_log, collect_checkpoints
```

## 5. IA & URL (프로그매틱/에디토리얼 분리)
| | A 경로 | B 경로 |
|--|--|--|
| 시세(팩트) | `/[cat]/[region]/[item]` | `/item/[item]` |
| 지역허브 | `/[cat]/[region]` | — |
| 카테고리 | `/[cat]` | `/[cat]` |
| 시점 | `/price/[date]` | 동일 |
| 비교 | 지역 vs 전국 | `/compare/[A]-vs-[B]` |
| **대체재** | `/swap/[item]` (배추↔양배추 등) | 동일 [신규] |
| 시즌(에디토리얼) | `/season/[event]` | 동일 |
| 필러/에버그린 | `/guide/`, `/learn/` | 동일 |

- **분리 원칙**: 시세 팩트 페이지(데이터)와 시즌·가이드(AI 에디토리얼)를 URL·템플릿으로 구분 → AI Overview/제로클릭 리스크를 팩트 페이지에 전이시키지 않음.
- **Cannibalization 맵**: item=품목 추이/타이밍, region=지역비교, compare=품목간, swap=대체재, season=시즌집약 — 주 키워드 의도 비중복 사전 매핑.

## 6. 페이지 = 제품 (도구 + 정정/신설)
- **구매 타이밍 진단**: 현재가 vs 평년/전년 → "평년 대비 낮음/높음 + 추세"(**서술형, 지시형 금지**)
- **데이터 기반 저점 시즌 위젯** [정정]: "제철=저점" 단정 폐기 → **과거 시계열에서 실제 가격 저점 월**을 산출해 표시(제철과 다를 수 있음)
- **대체재 추천** [신규]: 해당 품목 강세 시 가격 낮은 대체재 제시 — 제로클릭 불가 + CPA 연결 + 도메인 고유
- **원/kg 환산 비교기** (weight 한정), **시계열 차트**(결측 명시), **도소매 마진 게이지**(매칭 한정)

조건부 섹션:
```
if (현재가 > 평년*1.2)   → "최근 강세" + 대체재 제안
if (현재가 < 평년*0.85)  → "평년 대비 저렴" (서술) + (요율통과 시)CPA
if (시즌 D-30 이내)      → 시즌 허브 링크
if (unit_type==='weight')→ 원/kg 비교
if (도소매 매칭)         → 마진 섹션
if (is_discount)         → 할인가 혼입 고지
```

## 7. 콘텐츠 3-Layer + 시즌 엔진
- L1(80%): 품목(×지역) 시세·타이밍 (도구)
- L2(15%): 필러 — "김장 배추 언제 사야 쌀까", "삼겹살 비쌀 때 대체", 제철·대체재 가이드
- L3(5%): 에버그린 — "평년가격이란", "등급(상/중/하) 차이", "농산물 가격 읽는 법"
- 시즌 엔진: `season_calendar` D-21~30 자동 선발행 (에디토리얼 영역)

## 10. 수익화 (조건부·단위경제 점검)
1. **CPA — 조건부 1차** [수정]: 신선식품(쿠팡파트너스 등) 요율이 페이지 단가를 넘기면 1차. **요율은 얇은 식료품 마진 탓에 낮을 수 있음 → Gate 0에서 실측 후 결정.** 미달 시 AdSense 폴백.
2. **AdSense — 폴백/2차**: 도구로 체류 확보된 페이지 한정. 한국 RPM 낮음 전제.
3. **단위경제 점검 [신규]**: (예상 트래픽 × RPM/CPA) vs (호스팅+데이터비). 특히 A-paid는 데이터구매비 회수 가능선 명시. 트래픽이 김장·명절에 **lumpy** → 비수기 바닥 견디는지 확인.
4. audience: 소매=수익화 자산 / 도매=B2B 약함(허브·내부링크 역할).
5. **금지**: "정부 vs 마트" 비교, 가격 예측 단정, "지금 사세요" 지시형+제휴(표시광고법). 서술형만.

## 12. 사전 커밋 Kill 기준 & 게이트 순서
**순서**: Gate 0.5(API+라이선스 프로브) → Gate 0(DataLab+요율+단위경제) → 빌드

| 게이트 | 항목 | Kill / 분기 |
|--------|------|-------------|
| 0.5 | **라이선스(게이트②)** | 상업 재배포 불가 → **전면 kill** / KADX 재배포 불가 → A-paid 탈락 |
| 0.5 | **지역 차원(게이트①)** | 무료有→A-free / 유료만→A-paid후보 / 없음→B |
| 0.5 | 소매↔도매 조인 | 매칭률 낮음 → 마진 제거 |
| 0.5 | unit 분류 | 개수 기반 과반 → 원/kg 가치 반감 |
| 0.5 | 등급·결측·할인·백필페이징 | 행수 폭발/결측 과다 → 표기·용량 정책 흡수 |
| 0 | **DataLab(평시+시즌+피벗의도)** | "언제 싸나/추이/제철" 검색량 미달 → kill (피벗이 수요 해결 못 함) |
| 0 | **CPA 요율 실측** | 식료품 요율 미미 → CPA 1차 폐기, AdSense 단독 재계산 |
| 0 | 단위경제 | (트래픽×수익) < 비용 → kill / A-paid면 강등 |
| 0 | 제로클릭 성향 | 시간·계절 의도까지 즉답형 일색 → 재고 |
